## 3. 优先级规划

### P0 - 高优先级（必须完成）

#### 3.1 PeerConnection抽象层
**目标**: 简化通话创建流程，将复杂的底层模块封装为简单的API

**当前问题**:
- 用户需要手动创建Stream/Track/Transport等多层对象
- ICE/DTLS/SRTP需要手动协调
- 通话建立流程复杂

**设计方案**:
```
┌─────────────────────────────────────────┐
│            PeerConnection               │
│  (统一的通话抽象，封装所有底层细节)       │
├─────────────────────────────────────────┤
│  - CreateOffer() / CreateAnswer()       │
│  - SetLocalDescription()                │
│  - SetRemoteDescription()               │
│  - AddTrack() / RemoveTrack()           │
│  - OnIceCandidate()                     │
│  - OnTrack()                            │
└─────────────────────────────────────────┘
```

**预期工作量**: 3周
- 设计接口: 3天
- PeerConnection核心实现: 10天
- 状态机管理: 3天
- 与现有模块对接: 4天

**产出物**:
- `peer_connection.h` / `peer_connection.cpp`
- 对应单元测试 (10个)
- 集成测试 (2个)

---

#### 3.2 媒体同步机制
**目标**: 实现RTP时间戳与NTP同步、唇音同步

**当前问题**:
- 没有NTP时间戳同步机制
- 音视频可能不同步
- 播放端无法精确控制播放时间

**设计方案**:

```
┌─────────────────────────────────────────────────────┐
│              MediaSynchronizer                      │
├─────────────────────────────────────────────────────┤
│  1. NTP/RTP映射                                      │
│     - 解析RTCP Sender Report (SR)                   │
│     - 建立NTP ↔ RTP对应关系                          │
│     - 计算timestamp offset                          │
│                                                     │
│  2. 唇音同步 (Lip Sync)                             │
│     - 视频帧携带RTP timestamp                        │
│     - 音频帧携带RTP timestamp                        │
│     - 根据timestamp差值同步播放                      │
│                                                     │
│  3. 播放时间控制                                     │
│     - 调整播放速率                                   │
│     - 插帧/丢帧策略                                  │
└─────────────────────────────────────────────────────┘
```

**预期工作量**: 2周
- RTCP SR解析: 3天
- NTP/RTP映射计算: 3天
- 唇音同步逻辑: 5天
- 测试验证: 3天

**产出物**:
- `media_synchronizer.h` / `media_synchronizer.cpp`
- 对应单元测试 (8个)
- 同步机制验证测试 (2个)
